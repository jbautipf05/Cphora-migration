# H-003 — Modal "Nuevo lead" completo + variante institucional · P0

## 1. Diagnóstico (Demo6 vs React)
Demo6 (`openLeadForm` 4073): Tipo en **radio** (Lead / Lead institucional), campos
institucionales **Razón Social / NIT / Contacto**, **editor de Productos de interés**
(buscador por nombre + cantidad + "+ Agregar producto") y **Valor estimado auto/readonly**
= Σ(precio×cantidad) (`leadValorFromProductos`). React tenía: Tipo como `<select>`,
institucional con sólo Razón Social/NIT (faltaba **Contacto**), **sin** editor de
productos, y Valor como input **manual**.

## 2. Cambio aplicado (`src/views/Leads.jsx`)
- **Tipo → radios** "Lead / Lead institucional" (espejo Demo6).
- Institucional: Razón Social, NIT, **+ Contacto** (agregado), condicionales.
- **`ProductPicker`** (componente nuevo en el archivo): buscador de producto con
  **autocompletado** (filtra por nombre, dropdown, máx 12) — elegido sobre un
  `<select>` porque el catálogo puede tener cientos de productos (decisión del usuario).
- **Editor "Productos de interés":** filas con ProductPicker + cantidad + quitar (×)
  y botón "+ Agregar producto". `form.productosInteres = [{productId, qty}]`.
- **Valor estimado auto/readonly:** `leadValor = Σ(product.price × qty)`, recalculado
  en vivo; input readonly (reemplaza el manual).
- `save()` persiste `productosInteres` + `valor` calculado.
- `*` agregados en Tipo/Clasificación/Canal/Estado (diferidos de H-004) → parity completa.
- **Fix:** los errores inline (H-004) ahora se **resetean** al abrir el modal
  (nuevo/editar/intent) — antes quedaban pegados de un intento previo.
