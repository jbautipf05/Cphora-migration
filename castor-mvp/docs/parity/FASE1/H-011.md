# H-011 — Form de cotización "deficiente" (re-auditoría vs Bloque 1) · P0

El usuario reportó el form de cotización como "mal hecho" sin especificar. El
inventario nota que **se solapa con el Bloque 1** (ya reconstruido y verificado).
Se hizo una **re-auditoría campo por campo** Demo6 ↔ `NuevaCotizacionModal` (en vivo,
Preview MCP) para cazar cualquier diferencia restante.

## 3. Cambio aplicado
- **`src/components/Modal.jsx`**: se agregó el tamaño `'3xl' → max-w-3xl` (adición no
  rompe nada).
- **`src/components/NuevaCotizacionModal.jsx`**: `size="lg"` → `size="3xl"` para igualar
  el ancho de Demo6 (`max-w-3xl`).
- (Título h2/700 vs h3/600 = **A26**, que el usuario decidió **no corregir** →
  `MEJORAS_PROPUESTAS.md`. No se tocó.)

## 4. Verificación (en vivo, Preview MCP)
- Modal de cotización ahora `max-w-3xl` (= Demo6). ✅
- Labels, fila de ítem (4/4/1/2/1), resumen y botones idénticos a Demo6. ✅
- Buscador/autollenado/arrastre/lead nuevo/validaciones siguen OK (Bloque 1). ✅
- `eslint` 0 errores · `vite build` OK.

## 5. Edge cases
- Cubiertos por la verificación del Bloque 1 (ver `VERIFICACION_BLOQUE_1.md`): buscar
  inexistente, toggles por campo, lead nuevo, descuento>50, 0 ítems, etc.

## 6. Estado
✅ **Resuelto.** El form ya era fiel desde Bloque 1; la única diferencia real
restante (ancho del modal) quedó corregida. La queja "mal hecho" correspondía al
estado **previo** al Bloque 1.
