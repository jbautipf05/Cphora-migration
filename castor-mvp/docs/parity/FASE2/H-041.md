# H-041 — Modal "Nuevo producto" completo · P0 (GRANDE, 3 bloques)

**Módulo:** Lista de Precios → "+ Nuevo producto" → modal · **Prioridad:** P0

## Diagnóstico
Lista de Precios no tenía modal de alta. Demo6 lo divide en 3 bloques: A datos básicos + imagen,
B dimensiones, C ubicación/stock + botones.

## Decisión de diseño (reuso vs nuevo) — para revisión humana
El inventario pedía "evaluar con el usuario" la reutilización del modal de `Innovacion.jsx` (que ya
crea productos pero **con BOM, áreas y lifecycle**, enviándolos a auditoría). Dado que (a) el inventario
especifica un modal **simple sin BOM** para Lista de Precios y (b) el contexto difiere (alta rápida de
catálogo vs producto de I+D), se construyó un **modal dedicado simple** en `ListaPrecios.jsx`
(no se reutilizó el de Innovación). El selector AskUserQuestion no estaba disponible (glitch); decisión
reversible. **A confirmar en cierre de fase.**

## Cambio
`src/views/ListaPrecios.jsx` (+ `useApp().add/nextId`, `Modal`, form fields, `uid`):
- **Bloque A:** zona imagen (emoji/preview) + Nombre del producto* / SKU (autogenerado) / Categoría* (dropdown
  de categorías existentes) / Precio comercial* / Descripción.
- **Bloque B "Dimensiones":** Ancho / Alto / Profundidad (cm).
- **Bloque C "Ubicación y Stock":** Bodega/sede inicial* (dropdown bodegas terminado) + Stock inicial*.
- Botones: "Cancelar" / "Crear producto".
- `saveNewProduct`: `nextId('PROD','prod',0)`, SKU autogenerado si vacío, `add('products', {...})`
  (`verified:true`, dimensions, price). Si stock inicial > 0, crea registro `finishedStock` (`uid('FS')`)
  para que el catálogo/disponibilidad lo refleje.
