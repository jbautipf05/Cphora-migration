# H-019 — Consolidar columna PROCESOS · P2

**Módulo:** Clientes → tabla → columna PROCESOS · **Prioridad:** P2

## Diagnóstico
React mostraba 3 sub-conteos coloreados con barras (`{pedidos}/{cotizaciones}/{leadsVinc}`),
visualmente saturado. Demo6 (`Castor_Dashboard_Demo6.html:4317`) muestra un único entero,
alineado a la derecha, **dorado** si >0 (si no, muted).

## Cambio
`src/views/Clientes.jsx` — render de columna PROCESOS ahora:
`<span className={font-bold + (n ? 'text-gold-accent' : 'text-muted/50')}>{n}</span>`,
donde `n = summary[r.id].procesos` (= pedidos + cotizaciones + leads vinculados, el valor
consolidado que ya se calculaba). `align: 'right'` se mantiene.

## Verificación (Preview MCP, :5173)
- Celdas: `1`, `2`, `1`, `1`, `3` — número único, color `rgb(201,169,97)` (gold-accent), sin `/`.

## Edge / caveats
- Suma los mismos 3 sub-procesos que React ya rastreaba (lo pedido por H-019: "sumar los
  sub-procesos"). La definición de Demo6 `customerProcesosActivos` (Demo6:4229) además filtra
  por estado activo (excluye ganado/aceptada/entregado) e **incluye seguimientos pendientes**;
  esa semántica más fina afecta también el KPI "Con procesos activos" → se evaluará en H-016.
