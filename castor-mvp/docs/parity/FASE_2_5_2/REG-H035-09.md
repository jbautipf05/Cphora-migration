## 1. Síntoma
React aceptaba registrar una venta **sin pago** (todos los campos de pago vacíos).

## 2. Inspección de Demo6 (respuestas a las 5 preguntas)
`openSaleForm:5647-5664` + `saveSale:5975-6185`.

| # | Pregunta | Demo6 |
|---|---|---|
| 1 | ¿Venta sin pago (monto=0)? | **NO.** `payAmount` = `type="number" min="1" required` (`:5651`). Sin path de crédito. |
| 2 | Si monto>0, ¿obligatorios? | **TODOS** (`:5649` "Todos los campos son obligatorios"): monto≥1, método, cuenta receptora, referencia, soporte — los 5 `required` (`:5651-5661`). |
| 3 | ¿Soporte siempre o solo si monto>0? | **SIEMPRE.** `required` + backstop incondicional `saveSale:5985`. |
| 4 | ¿Monto ≤ total? | **No existe.** `saveSale:6169` solo capa el % a 100; no rechaza sobrepago. |
| 5 | ¿Múltiples métodos? | **NO.** Un único bloque de pago por venta. |

`recalcSale:5945` auto-rellena `payAmount = total·anticipo%/100` salvo edición manual.

## 3. Decisión
**FIEL A DEMO6** (confirmado por el usuario 2026-05-26 vía AskUserQuestion): los 5 campos obligatorios
SIEMPRE, monto≥1, sin venta a crédito (se elimina la flexibilidad previa de React). Sin tope monto≤total
(Demo6 no lo tiene). Un solo método (React ya era de un método). La lógica condicional-por-monto que se
había planteado como hipótesis NO aplica porque Demo6 no admite monto=0.

## 4. Implementación (patrón REG-06)
- `validatePayment(v)` → `errs` para `payAmount`(≥1), `payMethod`, `payBankId`, `payReference`, `paySoporte`.
  Monto efectivo = `payManual ? Number(payAmount) : suggestedPay` (anticipo%·total).
- `handleSubmit`: `errs = { ...validateClient, ...validateControl, ...validatePayment }`.
- `errLine` bajo los 5 campos; asterisco añadido a "Soporte de pago *" (faltaba).
- Limpieza al editar: método/cuenta/referencia/soporte usan `set` (ya limpia); `payAmount` y `% Anticipo`
  tienen `onChange` propio → se les agregó `setErrors` para limpiar `payAmount`.
- **Gates efectivos:** `payReference`, `paySoporte`, y `payAmount<1` (p.ej. anticipo 0 o monto manual vacío).
  `payMethod`/`payBankId` tienen default no vacío → defensivos (salvo que no existan cuentas bancarias).
- Aplica en venta normal **y** innovación (Demo6 también exige pago en ambos; el bloque se renderiza igual).
