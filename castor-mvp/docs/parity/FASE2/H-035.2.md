# H-035.2 — Modal "Nueva venta": Bloque Items dinámico · P0 (parte 2/3)

**Módulo:** Ventas → modal Nueva venta → bloque Items · **Prioridad:** P0

## Diagnóstico
El bloque de items era un producto único provisional (H-035.1). Demo6 (`Castor_Dashboard_Demo6.html:5605-5760`)
permite múltiples items con: radio Stock/Producción por ítem, PRODUCTO/CANTIDAD/PRECIO UNIT (readonly)/
% DESC/TOTAL, botón X para quitar, "+ Agregar item", texto informativo y textarea de notas por ítem.

## Cambio
`src/components/NuevaVentaModal.jsx`:
- `addItem()` / `removeItem(i)` sobre `f.items[]`.
- Bloque dinámico: título + "+ Agregar item"; texto "Puedes mezclar items de stock… con items de producción…".
- Por ítem: radios **Stock**(verde)/**Producción**(ámbar) → `it.tipo`; grid 12 cols con Producto* (dropdown),
  Cantidad, Precio unit. (readonly desde lista), % Desc., Total (`itemTotal` = precio×qty×(1−desc/100)),
  botón "×" rojo para quitar, y textarea de comentario.
- El total del modal (footer) suma todos los `itemTotal`.
- `submitSale` (Ventas) ya consume `items[]`; `tipo` del pedido = 'mixto' si mezcla, si no el del ítem.

## Verificación (Preview MCP, :5173)
- **E2E creación:** "+ Nueva venta" → nombre "Cliente Test E2E" → producto "Butaca Serenity" ($2.850.000) →
  Total modal $2.850.000 → "Registrar venta" → nueva fila **P-1254** en la tabla (Total $2.850.000,
  Saldo $1.425.000, 50%, PRODUCCION, OPs=1, +Pago). Filas 5→6. ✅
- **Multi-ítem:** "+ Agregar item" → 1→2 ítems, 2 botones "×". ✅
- Datos de prueba limpiados (localStorage), seed restaurado a 5 pedidos. eslint exit 0.

## Edge / caveats
- Precio unit. es readonly desde la lista (igual que Demo6); el descuento es por ítem.
- El bloque **Registro de pago + liquidación** (subtotales stock/producción, descuento, total) llega en H-035.3;
  por ahora el %Anticipo del control general define el pago inicial.
- Multi-ítem se persiste en `order.items[]`; `productId`/`qty` del pedido = primer ítem (compat con
  Produccion/Auditoria, que aún leen el modelo plano — parity de esos módulos es fase posterior).
