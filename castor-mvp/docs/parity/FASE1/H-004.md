# H-004 — Validaciones del formulario de Lead · P0

## 1. Diagnóstico (Demo6 vs React)
- **Demo6** (`openLeadForm`/`saveLead`, líneas 4073/4186): valida con `required`
  nativo en los campos con `*`: **Tipo, Nombre, Teléfono, Canal, Clasificación,
  Estado, Asesor** (Asesor con "— Elegir —"). **No** valida formato de teléfono
  (sin chequeo de dígitos), y `valor` es **auto/readonly** (de productos de interés).
  No exige Documento/Email/Ciudad/Dirección/Productos.
- **React (antes):** sólo bloqueaba por nombre vacío (toast). Teléfono aceptaba
  letras; Asesor tenía default fijo (no forzaba elección). Sin mensajes inline.

## 2. Cambio aplicado (`src/views/Leads.jsx`)
- `validate(form)` → errores por campo: **Nombre** (obligatorio), **Teléfono**
  (obligatorio · sin letras · ≥7 dígitos), **Asesor** (obligatorio).
- `save()` bloquea si hay errores (`setErrors` + toast) — no guarda hasta corregir.
- Errores **inline** (texto rojo) bajo Nombre/Teléfono/Asesor; se limpian al editar
  el campo (`set` borra el error de esa key).
- Labels marcados con `*`: Nombre *, Teléfono *, Asesor *.
- `EMPTY.asesor = ''` + opción "— Elegir —" en el select (espejo de Demo6).

## 3. Verificación (en vivo, Preview MCP)
- Guardar vacío → 3 errores inline (nombre/teléfono/asesor), **no** se crea el lead. ✅
- Teléfono "abc123"/"telefono123" → "El teléfono no puede contener letras", bloquea. ✅
- Teléfono "123" → "El teléfono debe tener al menos 7 dígitos", bloquea. ✅
- Nombre + Teléfono "300 555 1122" (con espacios) + Asesor → **crea OK** (espacios
  permitidos, no rompe el formato del seed). ✅
- `eslint` 0 · `vite build` OK. Screenshot de errores inline en el chat.

## 4. Screenshot antes/después

- **Antes:** sólo toast por nombre; teléfono con letras se guardaba.

## 5. Edge cases probados
- Vacío total → bloquea con 3 errores. ✅
- Letras en teléfono → bloquea. ✅
- <7 dígitos → bloquea. ✅
- Teléfono con espacios (formato seed) → válido. ✅
- Error se limpia al corregir el campo. ✅

✅ **Resuelto.**

## 7. Caveats / dudas
- **Interpretación de "solo dígitos":** lo implementé como **sin letras + ≥7 dígitos**
  (permite espacios/guiones/+), para no invalidar los teléfonos con espacios del
  seed ("300 555 1122"). Si querés **estrictamente** sólo dígitos (rechazar espacios),
  decímelo y lo ajusto (habría que normalizar también el seed).
- Las etiquetas `*` de los **selects** Tipo/Canal/Clasificación/Estado (que en React
  siempre tienen valor) las dejo para **H-003** (rebuild del modal), para no duplicar
  trabajo. Aquí marqué `*` sólo en los campos validados.
- `valor` auto/readonly + editor de "productos de interés" → corresponden a **H-003**
  (el inventario los lista ahí), no a H-004.
