## 1. Síntoma
En modo **Producción** faltaban los selectores de acabado (color madera/metal/tejido) y el buscador de
tela de tapicería; el modal no capturaba esos datos que Demo6 sí pide según el producto.

## 2. Inspección de Demo6 (`renderSaleAcabados:5841-5868`, constantes `:5521-5523`)
Solo en modo Producción, condicional por `product.areas`:
- `Preparación y pintura` → **Color madera** (`COLOR_MADERA`, 11 opciones incl. **'Otro'** → input libre).
- `Pintura electrostática` → **Color metal** (`COLOR_METAL`, 5).
- `Tapicería` → **Tela tapicería** (buscador autocompletado sobre `supplies` categoría Telas).
- `Tejido` → **Color tejido** (`COLOR_TEJIDO`, 19).
El contenedor se muestra como grid solo si el producto tiene ≥1 de esas áreas.

## 3. Implementación (fiel a Demo6 + los 4 acabados confirmados por el usuario, incl. Tejido)
- Constantes `COLOR_MADERA/METAL/TEJIDO` + `ACAB_AREAS` portadas de Demo6.
- Item shape extendido: `colorMadera, colorMaderaOtro, colorMetal, telaId, colorTejido`.
- `renderAcabados(it, i)`: devuelve los selectores condicionales por `areas`, **solo si** `tipo!=='stock'`
  y hay producto. 'Otro' en madera muestra el input libre `colorMaderaOtro`.
- **REG-02:** componente `TelaPicker` (reusa patrón `ProductPicker` de Leads): input + dropdown filtrado
  sobre `supplies` (`category` Telas/Textiles), mostrando `name · id · fmtCOP(cost)/unit`. Se pasa
  `supplies` como prop desde Ventas.
- `setItem`: al cambiar producto o pasar a Stock, **resetea los acabados** (dejan de aplicar).
- Reactivo por estado (sin handlers imperativos). Se renderiza entre el indicador de stock y el comentario.

## 4. Mapa producto→acabados (seed, para validar a mano en Producción)
