# B3 — `fmtCOP`/`fmtNum` duplicado (Fase 1.5)

**Tipo:** corrección técnica (deduplicación) · **Severidad:** 🟠 alta · **Origen:** REVISION_TECNICA_PRE_FASE2 §BLOQUEANTES / TD-26.

## Problema
`fmtCOP` (y `fmtNum`) existían **dos veces**, byte a byte idénticas, en `lib/format.js:11`
y `lib/accounting.js:42`. Fase 1 estandarizó las vistas comerciales en `format.js`, mientras
13 vistas (Clientes, ListaPrecios y todas las contables) seguían importando de `accounting.js`.
Dos fuentes que podían divergir → formateo de moneda inconsistente entre módulos.

## Cambios
Un solo archivo: `src/lib/accounting.js`.

```diff
- const _copFmt = new Intl.NumberFormat('es-CO', { style:'currency', currency:'COP', maximumFractionDigits:0 });
- export const fmtCOP = (n) => _copFmt.format(Math.round(n || 0));
- export const fmtNum = (n) => new Intl.NumberFormat('es-CO', { maximumFractionDigits:0 }).format(Math.round(n || 0));
+ // fmtCOP / fmtNum: fuente única en lib/format.js. Se re-exportan aquí por
+ // compatibilidad con las vistas contables que importan desde este módulo (B3 / TD-26).
+ export { fmtCOP, fmtNum } from './format';
```

- **Fuente única:** `lib/format.js` (sin cambios).
- **Compatibilidad:** las 13 vistas que hacen `import { fmtCOP } from '../lib/accounting'`
  siguen funcionando sin tocarlas (re-export).
- **Sin ciclo de dependencias:** `format.js` es standalone (no importa nada del proyecto);
  `accounting.js → format.js` es unidireccional.
- Se eliminó `_copFmt` local (quedaba huérfano al quitar `fmtCOP`/`fmtNum`).
