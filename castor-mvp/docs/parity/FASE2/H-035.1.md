# H-035.1 — Modal "Nueva venta": Bloque Cliente · P0 (parte 1/3 del modal grande)

**Módulo:** Ventas → "+ Nueva venta" → modal · **Prioridad:** P0

## Diagnóstico
El modal "Nuevo pedido" de React era un `FormGrid` plano (Cliente texto + Producto + pocos campos).
Demo6 (`Castor_Dashboard_Demo6.html:5530-5603`) abre un modal grande con un **bloque Cliente**:
buscador inteligente (4 píldoras Nombre/Celular/Correo/Documento + resultados), separador
"— o crear/editar —", formulario exprés 2 columnas, dirección de entrega con "📋 = Dirección",
y filtros de origen (MEDIO/PDV/CIERRE DE VENTA/ABONO A BANCOS).

## Cambio
- **Nuevo componente** `src/components/NuevaVentaModal.jsx` (Modal `size="lg"` = max-w-4xl) con el
  bloque Cliente completo:
  - Píldoras de campo de búsqueda (Nombre/Celular/Correo/Documento) — la activa en dorado.
  - Input "Buscar cliente existente…" → lista flotante de resultados (ID·Nombre · Ciudad), máx 8.
  - Al seleccionar: autocompleta nombre/doc/celular/correo/ciudad/dirección y fija `customerId`.
  - Separador "— o crear/editar —".
  - Exprés 2 col: Nombre*/Documento* · Celular*/Correo* · Ciudad*/Dirección*.
  - Dirección de entrega* full-width + botón "📋 = Dirección" (copia la dirección del cliente).
  - Origen: MEDIO*/PDV*/CIERRE DE VENTA*/ABONO A BANCOS*.
  - (Provisional en 1/3: Producto/Cantidad + control Asesor/Tiempo/Documento/%Anticipo — se reemplazan en H-035.2/.3.)
- `src/views/Ventas.jsx`: eliminado el modal inline antiguo + `emptyOrder`/`setF`/`formTotal`/`saveOrder`;
  nuevo `submitSale(data)` que resuelve/crea el cliente con datos completos (B2/ADR-011: `customerId` FK),
  ancla a `today()` (B1) y contabiliza con `postSale`. Botones "+ Nueva venta"/"+ Venta innovación" abren el modal.
- Imports limpios (quitado `FormGrid`). eslint exit 0.
