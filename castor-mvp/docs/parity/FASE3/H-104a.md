## Cambios aplicados (2 deltas de paridad con Demo6)
Sobre `src/views/Innovacion.jsx`, modal "Nuevo producto":

1. **Subtotal por fila del BOM** — cada insumo muestra `costo × qty` a la derecha (espejo de
   la columna `row-cost` de Demo6 `addBomRowMaster`). Antes solo se veía el total.
2. **Auto-relleno en vivo de "Costo final"** — el campo se actualiza con el total del BOM a
   medida que se editan insumos/cantidades, **hasta que el usuario lo edita manualmente**
   (flag `costTouched`); ahí deja de auto-actualizarse. Espejo de
   `recalcBomCostMaster` (Demo6:8488-8489). Antes React solo hacía ese fallback al guardar
   (`Number(form.cost) || bomCost(...)`), no en vivo.

## DECISIÓN SIN INPUT DEL CLIENTE
**Las 2 deltas se aplicaron solo al modal "Nuevo producto"** (el que pide el requerimiento),
no al modal de **edición** de ficha (master card), que hoy no tiene editor de BOM (sí tiene
costo/precio/margen manuales). Razón: el cliente pidió explícitamente el flujo de *alta* de
producto con BOM; editar el BOM de un producto existente es un caso secundario y de menor
valor para la demo. Queda anotado como posible extra futuro (**EX-F3-01**: portar editor de
BOM al modal de edición de Innovación). *Defensa en demo:* el alta de producto calcula el
costo desde el BOM en vivo, igual que Demo6.
