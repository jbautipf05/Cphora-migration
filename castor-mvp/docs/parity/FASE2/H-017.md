# H-017 — Barra de filtros con etiquetas y contenedor unificado · P2

**Módulo:** Clientes → barra de búsqueda y filtros · **Prioridad:** P2

## Diagnóstico
React tenía selectores flotantes (`<Toolbar>` + `SelectFilter`) sin etiquetas, con placeholders
largos y sin contenedor. Demo6 (`Castor_Dashboard_Demo6.html:4267-4276`): fila dentro de un
`panel`, grid de 6 columnas, cada filtro con `<label class="label">` (mayúsculas pequeñas),
Buscar ocupando 2 columnas con placeholder "Nombre, doc, teléfono...".

## Cambio
`src/views/Clientes.jsx` — reemplazado `<Toolbar>` por:
- `<div className="panel mb-5 p-4">` contenedor.
- `<div className="grid grid-cols-2 gap-3 md:grid-cols-6 mb-3">`.
- Buscar: `col-span-2`, `<label className="label">Buscar</label>` + `input.input-field`, placeholder "Nombre, doc, teléfono...".
- Ciudad / Asesor / Tipo / Estado: `<label className="label">` + `select.input-field`.
  - Tipo: opciones "Persona" (value `lead`) / "Institucional" (value `institucional`) — coherente con tipos internos de React.
  - Estado: "Activo"/"Inactivo"/"VIP".
- `ClearFiltersButton` debajo del grid ("× Limpiar filtros").
- Imports sin uso eliminados (`Toolbar`, `SearchBox`, `SelectFilter`, `EstadoBadge`, `IconSearch`); eslint en 0.

## Verificación (Preview MCP, :5173)
