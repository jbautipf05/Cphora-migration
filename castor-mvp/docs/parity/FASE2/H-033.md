# H-033 — Reestructuración de columnas de tabla Ventas · P0

**Módulo:** Ventas → tabla principal · **Prioridad:** P0
**Decisión de alcance:** COSMÉTICO para IDs (etiqueta `P-XXXX`, id real `PED-` intacto).

## Diagnóstico
React tenía columnas PRODUCTO, MEDIO, PDV, ESTADO (redondo) que saturaban. Demo6
(`Castor_Dashboard_Demo6.html:5285-5333`) usa: Pedido (P-XXXX), Cliente, Asesor, OPs, Total,
Saldo, Pago, Tipo, Doc, Vence.

## Cambio
`src/views/Ventas.jsx` — columnas reescritas:
- **ELIMINADAS**: Producto, Medio, PDV, Estado.
- **Pedido**: etiqueta `P-XXXX` dorado semibold (`id.replace(/^PED-/, 'P-')`, id real intacto) + badge "INN" si `tipo==='innovacion'`.
- **OPs** (nueva): badge gris centrado con `opsCount` (= `items.length` o 1 — modelo plano, hoy 1).
- **Total**: dorado-light semibold.
- **Saldo**: rojo (`red-400`) si >0, verde (`emerald-400`) si 0.
- **Pago**: `ProgressBar` con color por avance (verde #10b981 si 100%, azul #38bdf8 si parcial).
- **Tipo**: SE MANTIENE el badge React (azul Producción / verde Stock) — mejora a preservar del inventario.
- **Doc**: texto uppercase muted (como Demo6).
- **Vence** (nueva, reemplaza Estado): `dueDate` o derivada `addDays(orderDate, tiempo)`; badge `{d}d`
  rojo si vencido (<0), ámbar si ≤7, verde si >7, gris si saldado (paid≥100 / entregado / cancelado).
- Texto de ayuda sobre la tabla: "💡 Doble clic en una fila abre el detalle del pedido con sus OPs."
- **Apertura del detalle por doble clic** (Demo6:5323): añadido `onRowDoubleClick` a `DataTable`
  (cambio aditivo, no afecta otras vistas que usan `onRowClick`). Título del detalle también muestra `P-XXXX`.
- Imports sin uso eliminados (`TIPO_DOC`, `MEDIO_VENTA`, `PDV_VARIANT`). eslint exit 0.
